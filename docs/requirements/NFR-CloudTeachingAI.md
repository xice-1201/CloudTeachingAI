# CloudTeachingAI 非功能性需求文档（NFR）

**文档版本**：v1.1
**创建日期**：2026-03-14
**最后更新**：2026-03-16
**关联文档**：URD-CloudTeachingAI.md v2.4，FRD-CloudTeachingAI.md v1.2

## 1. 性能要求

### 1.1 响应时间

| 操作类型 | 目标响应时间 | 最大可接受时间 |
|----------|-------------|---------------|
| 页面加载（首屏） | ≤ 2s | ≤ 4s |
| 课程列表/搜索结果 | ≤ 1s | ≤ 2s |
| 视频首帧加载 | ≤ 3s | ≤ 5s |
| AI 智能助手首次响应 | ≤ 3s | ≤ 6s |
| 学习路线生成 | ≤ 5s | ≤ 10s |
| 作业提交确认 | ≤ 1s | ≤ 2s |
| AI 作业批改 - 简单作业（纯文字，≤ 1000 字） | ≤ 30s | ≤ 60s |
| AI 作业批改 - 复杂作业（长文档/文件上传/代码，> 1000 字或含附件） | ≤ 120s | ≤ 300s |
| AI 内容合规预审（单份资源） | ≤ 30s | ≤ 60s |
| AI 账号数据分析报告生成 | ≤ 10s | ≤ 20s |
| AI 平台数据分析报告生成 | ≤ 10s | ≤ 20s |

### 1.2 并发能力

- 系统支持至少 500 名用户同时在线使用
- 视频播放服务支持至少 200 路并发流
- 高峰期（如期末考试周）并发用户数可扩展至 1000+

### 1.3 文件上传

- 单个视频文件上传大小上限：2GB
- 单个文档/课件文件上传大小上限：100MB（含 PDF、DOC、DOCX；PPT/PPTX 上限为 200MB，因幻灯片含大量图片时常超 100MB）
- 作业文件上传大小上限：50MB
- 支持断点续传，避免大文件上传中断后重新上传

## 2. 可用性要求

### 2.1 系统可用性

- 系统年可用性 ≥ 99.5%（即年停机时间 ≤ 43.8 小时）
- 计划内维护窗口安排在非教学高峰期（如凌晨 2:00–4:00），提前 24 小时通知用户
- 核心功能（课程访问、作业提交）不可用时间每月 ≤ 4 小时

### 2.2 故障恢复

- 系统故障后恢复时间目标（RTO）≤ 2 小时
- 数据恢复点目标（RPO）≤ 24 小时（即最多丢失 24 小时内的数据）；作业提交、学习进度等关键操作数据实时写入，故障后可从最近一次成功写入点恢复，实际数据丢失窗口远小于 24 小时
- 视频播放中断后支持自动重连，不丢失播放进度

### 2.3 浏览器兼容性

支持以下桌面浏览器最新两个主要版本：
- Google Chrome
- Mozilla Firefox
- Microsoft Edge
- Safari（macOS）

不支持 Internet Explorer。

## 3. 安全要求

### 3.1 身份认证

- 用户密码使用 bcrypt 或同等强度算法加密存储，禁止明文存储
- 会话 Token 使用 JWT，设置合理过期时间（普通会话 2 小时，"记住我" 7 天）
- 支持 HTTPS 全站加密传输，禁止 HTTP 访问

### 3.2 访问控制

- 严格按角色控制数据访问权限，学生只能访问自己的学习数据
- 导师只能访问已建立关系且学生明确授权的学习档案
- 课程资源访问与选课状态绑定，未选课学生无法访问课程内容
- API 接口需验证请求方身份，防止越权访问

### 3.3 数据隐私

- 学生个人信息、成绩、能力图谱等敏感数据符合数据隐私保护相关法规
- 学生知识图谱和学习路径数据仅对本人及已授权导师可见
- 导师视角下，学生手机号、身份证号、家庭住址等个人身份信息不可见；作业原始提交内容不对导师开放，仅展示评分和评语
- 平台不向第三方共享用户个人数据

### 3.4 内容安全

- 用户上传文件需进行病毒扫描
- 讨论区内容支持关键词过滤，防止违规内容发布
- 管理员可对违规内容进行下架处理
- AI 预审结果仅作为辅助标记，不自动触发下架，最终决定由管理员执行，防止误判影响正常教学内容
- AI 预审准确率要求：违规内容召回率 ≥ 90%，误判率（正常内容被标记为违规）≤ 5%
- 管理员对 AI 标记内容的人工复核响应时间 ≤ 24 小时

### 3.5 防攻击

- 登录接口防暴力破解：连续失败 5 次锁定 15 分钟
- 关键接口（登录、注册、文件上传）实施请求频率限制
- 防止常见 Web 攻击：XSS、CSRF、SQL 注入

## 4. 可扩展性要求

### 4.1 用户规模扩展

- 系统架构支持水平扩展，可通过增加服务器节点应对用户增长
- 初期支持 1000 名注册用户，架构设计支持扩展至 10000+ 用户

### 4.2 课程资源扩展

- 存储系统支持弹性扩容，初期存储容量 1TB，可按需扩展
- 视频转码和存储服务支持独立扩展

### 4.3 AI 智能体扩展

- AI 智能体服务与主平台解耦，支持独立部署和扩展
- 支持后续接入新的智能体功能模块，不影响现有功能

---

## 5. 可维护性要求

### 5.1 日志与监控

- 系统记录关键操作日志：用户登录、文件上传、作业提交、AI 调用
- 系统记录用户行为数据：页面访问频率、功能模块使用次数、操作路径，供 AI 智能体分析功能接受度
- 日志保留周期 ≥ 90 天
- 提供系统健康监控面板，实时展示服务状态、响应时间、错误率
- 管理员可查看 AI 生成的平台使用分析报告，报告按需生成，不影响系统正常运行性能

### 5.2 数据备份

- 数据库每日自动备份，备份文件保留 30 天
- 视频等大文件存储采用冗余存储，防止单点故障导致数据丢失

### 5.3 部署与更新

- 支持不停机滚动更新，避免更新期间影响用户使用
- 前后端分离部署，支持独立更新

---

## 6. 易用性要求

### 6.1 操作效率

- 核心操作（找课程、看视频、提交作业）在 3 步以内完成
- 页面信息层次清晰，不同角色看到与自己相关的内容
- 关键操作（提交作业、发布课程）提供操作确认和成功反馈

### 6.2 错误提示

- 表单验证错误需给出明确的中文提示，指出具体问题
- 网络异常或操作失败时，给出友好提示并提供重试选项
- 文件上传格式或大小不符时，提前提示而非上传后报错

### 6.3 通知及时性

- 作业提交确认为同步响应，用户提交后页面立即（≤ 1s）返回成功状态，不经过异步通知队列
- 作业批改完成、教师回复等关键事件的站内通知延迟 ≤ 5 分钟
- 站内通知在用户下次登录时必须可见

## 7. 国际化与本地化

- 平台界面语言：简体中文（主要）
- 日期时间格式：北京时间（UTC+8）
- 文件编码：UTF-8
